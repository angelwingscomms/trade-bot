These four pieces are the core of the paper's feature engineering, and they are designed to work together in a volume-agnostic 1-minute pipeline. Here is how to implement each exactly as the paper describes, with code you can drop into your data builder.

## 1. Log vs standard returns as targets

The paper states: "Log returns and standard returns are identified as the most effective target variables for directional forecasting", and "Log Returns normalizes price changes and mitigates heteroscedasticity".

**Implementation logic**

- Do not predict raw price. Predict the next-bar change.
- Use log returns for training stability, keep standard returns for PnL accounting.

Formulas:
- Standard return: $r^{std}_t = \frac{C_t - C_{t-1}}{C_{t-1}}$
- Log return: $r^{log}_t = \ln\left(\frac{C_t}{C_{t-1}}\right)$

For your 3-class classifier:
1. Compute next-bar log return: `y_log = log(close.shift(-1) / close)`
2. Define thresholds by volatility, not fixed pips. The paper uses ATR-based risk, so mirror that:
   - `thr = 0.15 * ATR_14 / close` (about 15% of one ATR)
3. Label:
   - `+1` if y_log > thr
   - `-1` if y_log < -thr
   - `0` otherwise

```python
import numpy as np
import pandas as pd

df['ret_std'] = df['close'].pct_change()
df['ret_log'] = np.log(df['close'] / df['close'].shift(1))
df['atr'] = df['high'].sub(df['low']).rolling(14).mean() # simplified

thr = 0.15 * df['atr'] / df['close']
y_next = np.log(df['close'].shift(-1) / df['close'])

df['target'] = 0
df.loc[y_next > thr, 'target'] = 1 # up
df.loc[y_next < -thr, 'target'] = 2 # down (or -1 if you use signed labels)
```

Why log: it makes a 0.5% move up then down symmetric, and it compresses large moves so the EmbTCN-Transformer does not chase outliers during backprop.

## 2. EMA ratios for mean-reversion (binary +1/-1 only)

The paper: "EMA ratios provide a robust feature set for identifying mean-reversion opportunities", and the ratio "indicates the extent of price deviation from the trend mean".

You want only the sign, not the magnitude.

**Implementation**

1. Pick a slow trend EMA that matches 1m scalping, the paper uses EMA 20-50 for fast regimes.
2. Compute ratio: `ema_ratio = close / ema - 1`
3. Binarize: `regime = +1` if price above EMA, `-1` if below.

```python
ema_fast = df['close'].ewm(span=20, adjust=False).mean()
ema_slow = df['close'].ewm(span=50, adjust=False).mean()

# mean-reversion signal: deviation from slow EMA
df['ema_dev'] = df['close'] / ema_slow - 1

# binary regime only
df['mr_regime'] = np.where(df['ema_dev'] > 0, 1, -1).astype(np.int8)
```

Interpretation the model learns:
- `+1` = price is stretched above mean, higher probability of short-term pullback
- `-1` = price is below mean, higher probability of bounce

Do not feed the raw ratio. The binarization forces the network to treat mean-reversion as a regime, not a linear predictor, which matches the paper's intent to avoid overfitting to exact distances.

## 3. Normalized outlier transforms (sqrt / cubic root)

The paper: "handle outliers through 'normalized outlier' approaches — such as using square or cubic roots of the distance between an outlier and a threshold — which retains the significance of the extreme event without skewing the model's weight distribution".

**Implementation**

Do this on returns, not prices.

1. Compute rolling threshold: 3.5 × rolling MAD or std of log returns (last 500 bars)
2. For each bar where |r| > threshold:
   - distance = |r| - threshold
   - transformed = sign(r) × (threshold + distance^0.5) for sqrt
   - or sign(r) × (threshold + distance^(1/3)) for cubic

```python
r = df['ret_log']
rolling_std = r.rolling(500, min_periods=100).std()
thr = 3.5 * rolling_std

dist = (r.abs() - thr).clip(lower=0)

# choose one: sqrt preserves more, cubic compresses more
df['ret_log_norm'] = np.where(
    r.abs() > thr,
    np.sign(r) * (thr + np.sqrt(dist)),
    r
)

# cubic version
# df['ret_log_norm'] = np.where(r.abs() > thr,
# np.sign(r) * (thr + np.cbrt(dist)), r)
```

Why this works: clipping to zero kills information about flash moves. The root transform keeps the direction and rank of the outlier, but shrinks a 5-sigma move to about 2.2-sigma in sqrt space, so the Transformer does not allocate all capacity to rare spikes.

Apply the same transform to any other fat-tailed feature (Bollinger width, ATR spikes).

## 4. Hard regime filtering as features, not trade gates

The paper originally describes gating: "features from higher timeframes (5-minute, 15-minute, or 1-hour) act as filters for the 1-minute execution engine" and "bot may only be permitted to take long scalps if price is above the 200-period SMA on the 15-minute chart".

You want the opposite: keep the information, but let the model decide.

**Implementation**

1. Compute higher-timeframe regimes on resampled data
2. Forward-fill them to every 1m bar
3. Feed as binary features, do not block trades in code

```python
# resample to 5m and 15m
df_5m = df.resample('5min').agg({'close':'last'})
df_5m['ema50'] = df_5m['close'].ewm(50).mean()
df_5m['ema200'] = df_5m['close'].ewm(200).mean()
df_5m['regime_5m'] = np.where(df_5m['ema50'] > df_5m['ema200'], 1, -1)

df_15m = df.resample('15min').agg({'close':'last'})
df_15m['sma200'] = df_15m['close'].rolling(200).mean()
df_15m['regime_15m'] = np.where(df_15m['close'] > df_15m['sma200'], 1, -1)

# map back to 1m
df['regime_5m'] = df_5m['regime_5m'].reindex(df.index, method='ffill')
df['regime_15m'] = df_15m['regime_15m'].reindex(df.index, method='ffill')
```

Now your feature vector for each 1m bar contains:
- `mr_regime` (1m mean-reversion)
- `regime_5m`
- `regime_15m`

The EmbTCN-Transformer sees all three as inputs. During training it learns that longs in a `-1, -1, -1` regime have low expectancy, without you hard-coding a rule. This is what the paper means by "hard regime filtering" in the feature space: the regimes are hard (+1/-1), but filtering is soft and learned.

**Putting it together**

Your final `n_features` for the model becomes:
- normalized log return
- normalized outlier return
- mr_regime
- regime_5m
- regime_15m
- plus any timing features (high-low timestamp delta)

All are stationary, bounded, and free of volume data, exactly matching the paper's volume-agnostic design.
